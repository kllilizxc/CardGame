# 卡牌设计规范（Draft）

> 目标：给所有后续卡牌设计提供统一的“强度天花板”和“资源兑换”基准，避免出现毫无代价的过强卡牌。
>
> 本规范优先约束 **资源兑换 / 卡差获取 / 条件与代价**，数值曲线详见 `CARD_GENERATION_GUIDE.md` 与 `combat-baseline.json`。

---

## 1. 资源与“收益”的统一定义

在本游戏中，以下都视为“资源”或“实质性收益”：

- **手牌**：
  - 抽 1 张牌 = +1 资源。
  - 从牌库检索并加入手牌 = +1～+1.5 资源（比普通抽牌更精准）。
- **场上单位/持续卡**：
  - 召唤/生成 1 个可攻击/可挡伤的单位 ≈ +1 资源以上（视星级而定）。
  - 放置 1 张持续符箓/法器/法阵，且可在多个回合提供收益，也按 **≥1 资源** 计算。
- **弃牌堆可再利用的卡**：
  - 把牌从牌库/手牌送入弃牌堆本身不是纯负收益，如果后续可以被回收/复活，也需要视为“潜在资源”。
  - 从弃牌堆将卡牌返回手牌/场上，按来源视为 **+1～+2 资源**。
- **生命值与伤害**：
  - 生命值、伤害视具体数值与节奏进行单独平衡，不在本规范中给出固定“换算率”，但：
    - **直接伤害 + 抽牌/检索** 这类“既打血又赚牌”的效果，一律按高危设计处理。

> 简化口径：**“有没有额外牌（手牌、场上、可再利用的弃牌资源）凭空多出来？”** —— 只要答案是“有”，就属于需要重点审查的资源收益。

## 2. 基本设计原则

### 2.1 单卡不应轻松赚取大量卡差

- 普通卡牌（3 星及以下）在常规条件下：
  - **不应同时获得 ≥2 张以上的净牌差**（包括手牌、场面单位、可回收资源）。
  - 例如：
    - 抽 2 + 召唤 1 = 3 张显性资源，属于超标效果，只能放在 **极高费用/极高条件/副作用极大** 的设计中。

- 中高星卡（4–6 星）：
  - 在**无额外费用/献祭/弃牌/扣血**的前提下，
  - 建议上限：**+1 张牌差 + 若干数值或一次性伤害/治疗**。

- 7 星及以上（尤其是 BOSS 级/剧情卡）：
  - 可以设计 **+2 张牌差** 级别的效果，但必须同时满足：
    - 极难登场（多重献祭 / 特定组合 / 一局一次）。
    - 或附带 **高额代价**（自残、弃大量牌、锁定回合操作等）。

### 2.2 强力效果必须附带明确代价

当卡牌具备以下任一特征时，必须在设计上增加“刹车”：

- **高质量检索 + 直接上场/加入手牌**（如：从牌库选任何灵兽并直接召唤）。
- **大范围控制**（全场沉默、全场冻结、多单位破坏）。
- **资源回流**（从弃牌堆/牌库外大量回收资源）。
- **节奏扭转**（一次性清场 + 抽牌 + 回复生命）。

推荐代价形式：

- 需要消耗或献祭场上单位。
- 需要弃手牌、弃牌库顶、失去生命值。
- 使用后本回合/下回合受到行为限制（不能再攻击/不能再用某类卡）。
- 一场战斗仅能使用一次。

### 2.3 条件越容易达成，效果越要保守

- **无条件、随时可用**：效果应当偏弱或中规中矩。
- **带简单前置条件**（如“场上有灵兽”“你控制一张法阵”等）：可以稍强，但仍应控制在 **+1 资源级别**。
- **复杂或多段条件**（如“献祭2单位 + 弃2牌 + 特定场地”）：可以给到 **爆发级/终结级** 效果。

## 3. 不同卡牌类型的强度边界

### 3.1 灵兽卡

- 普通灵兽：
  - 以攻击力/生命值为主，附带 0–1 条中等效果。
  - 效果多为：小幅自增益、小幅抽牌/检索（通常仅限于 **抽1 或 检索1**）。

- 高星灵兽（5 星及以上）：
  - 需通过献祭或高费用才能登场。
  - 可以：
    - 登场时获得 **1 张牌差**（例如：检索1、复活1）。
    - 或者提供强力持续光环，但不再额外赚牌。
  - 避免“登场就清场 + 抽多牌”的复合超强效果。

### 3.2 法器卡

- **装备型法器**：
  - 提供持续攻击/防御增益，视为中长期价值。
  - 一般不再额外提供稳定抽牌，以免成为“永动机”。
  - 若附带抽牌/检索，应当：
    - 触发频率有限（例：每回合最多触发 1 次）。
    - 或有触发门槛（例：击杀单位时才抽牌）。

- **场上持续型法器**：
  - 可以提供范围性增益或减益，但数值需要明显弱于一次性大招。
  - 尽量避免既提供高额战斗力，又在资源上持续获利的“双重滚雪球”设计。

### 3.3 符箓卡

- 一次性强效果卡一般只有 1 次机会，因此可以略微放宽数值，但：
  - 单张符箓同时做到 **抽多牌 + 清场 + 回收** 属于禁止组合。
  - 推荐单次集中在一个方向：
    - 纯伤害 / 纯控制 / 纯检索 / 纯回复，中间可以点缀一点小量副效果（例如“造成伤害并抽1”）。

- 带持续效果的符箓，更接近“持续类法器”，应遵守法器的强度规则。

### 3.4 丹药卡

- 丹药本身不占卡组（或有独立槽位），容易被频繁使用，因此效果要相对保守。
- 常规丹药：
  - 只做一件事：回复生命 / 提升某单位攻击力本回合 / 驱散负面。
- 高阶丹药：
  - 多半需要稀有度限制、数量限制，或严重副作用（例如：本局不能再使用其他丹药）。

### 3.5 法阵卡

- 法阵改变的是“规则级环境”，一旦过强会导致对局单一化。
- 设计时控制在：
  - 提供 **1–2 项中等强度的被动效果**。
  - 若有抽牌/检索，就尽量不再给伤害/清场类被动。
  - 效果对双方都有效，效果只筛选特定类型单位，不筛选玩家。

## 4. 资源兑换范式（示例约束）

以下为设计时可参考的“范式”，不是硬性公式，但超出时需要特别论证。

### 4.1 抽牌 / 检索

- 正常费用下：
  - 抽 1：可以顺带一个小效果（+攻击力、+少量护盾）。
  - 抽 2：通常应有明显代价（弃 1、失去生命、消耗场面等）。
  - 抽 3 及以上：仅允许出现在 **极高星级 / 一局一次 / 马上结束游戏级** 的效果里，且卡片类型有所限制。

- 检索：
  - 检索任意卡 > 检索特定类别卡 > 检索限定标签卡。
  - 越精准，越接近强度上限，越需要条件与代价。

### 4.2 弃牌堆交互

- 从弃牌堆 **回手**：
  - 恢复的是“未来资源”，强度很高。
  - 单张卡正常上限为：回收 1 张 + 小幅附加效果（例如加一点攻击）。

- 从弃牌堆 **直接上场**：
  - 等同于“白赚一个单位/法器”，需要献祭、弃牌等明显代价。
  - 多个单位同时复活属于终极级大招。

### 4.3 清场与控制

- 单体移除：
  - 可以附带小额抽牌/治疗。
- 多体/全场移除：
  - 不再附带抽牌/检索等明显资源收益。
  - 若必须给副收益，则伤害/移除强度需大幅削弱。

## 5. 强力卡牌的“刹车装置”

设计明显强力的效果时，至少选择其中 1–2 项作为强制制动：

- **高费用 / 高星级 / 多重献祭**。
- **限定次数**：一回合一次 / 一局一次。
- **负收益绑定**：
  - 自己弃牌 / 失去生命 / 失去场面。
  - 使用后跳过抽牌、不能攻击、不能再使用某类卡。
- **条件苛刻**：需要特定标签、场景（如必须有法阵或特定单位）。

> 简单说：**凡是“看起来一张牌做了三四件好事”的设计，都必须有巨大代价或极低可达成率。**

---

## 6. 文本与实现上的注意事项

- 卡牌文本要明确：
  - 资源的流向（从哪来、到哪去）。
  - 触发时机（回合开始/结束、攻击时、死亡时）。
  - 次数限制（每回合/每局/仅一次）。
- 设计时尽量避免：
  - 模糊的“随机大量收益”。
  - 难以用代码精确实现或高计算成本的效果。

---

## 7. 使用方式

- 本规范用于：
  - 设计新卡前的**自检**与**评审对话基础**。
  - 帮助 AI 生成卡牌时设定“强度上限提示”。
- 若后续实测发现某类效果普遍偏弱/偏强，可在本规范中：
  - 标注“修订记录”，
  - 对特定段落（如抽牌上限、弃牌堆回收上限）进行微调。
