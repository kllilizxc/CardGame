# HOPI Decisions

- Code changes may be automated inside a Goal; deploy/release requires explicit human approval.
- Story system first iteration will be a generic JSON-driven non-combat runtime, not a one-off hardcoded Phaser scene and not only an extension of the current expedition event/shop prototype.
- Story content conditions and effects should be declarative JSON plus TypeScript validation/evaluation; do not allow executable scripts in story content for the first iteration.
- Story system first iteration stays local/offline and ships with a small example story; authoring tools, AI prose expansion, and broad content production are deferred until the vertical slice works.
- During the story vertical-slice phase, `Boot` may temporarily start `StoryScene` directly for fast manual smoke; this was a temporary development shortcut, not a long-term navigation decision.
